왜 AngularJS인가? (Why AngularJS?)

HTML는 정적 문서를 선언하기엔 큰 장점을 가진다. 하지만 웹 어플리케이션에서 동적인 뷰를 사용하려면 앞서 말한 장점들이 모두 날라가 버린다. AngularJS는 웹 어플리케이션에서 HTML의 표현의 범위을 확장해 준다. 그 결과 놀라울 정도의 표현력으로 읽기 쉬우며 빠른 개발이 가능한 환경을 제공한다.

HTML is great for declaring static documents, but it falters when we try to use it for declaring dynamic views in web-applications. AngularJS lets you extend HTML vocabulary for your application. The resulting environment is extraordinarily expressive, readable, and quick to develop.

대체할 만한 프레임워크 (Alternatives)

다른 프레임워크들은 HTML, CSS 그리고 자바스크립트의 추상화하거나 DOM을 관리하는 방법을 제공하여 HTML 단점을 보완하고자 한다. 위 두가지 방법 모두 동적인 뷰를 위해 설계되지 않은 HTML의 근본적인 문제점을 가지고 있다.

Other frameworks deal with HTML’s shortcomings by either abstracting away HTML, CSS, and/or JavaScript or by providing an imperative way for manipulating the DOM. Neither of these address the root problem that HTML was not designed for dynamic views.

확장 가능성 (Extensibility)

AngularJS은 어플리케이션 개발에 최적의 프레임워크를 구축하기위한 도구 세트이다. 그것은 완벽하게 확장 가능하며, 다른 라이브러리와 함께 사용해도 잘 작동한다. 모든 기능은 당신 자신의 개발 흐름과 요구에 맞추어 변경하거나 교체 할 수 있다. 그 방법을 알기 위해 계속 읽어 보면 된다.

AngularJS is a toolset for building the framework most suited to your application development. It is fully extensible and works well with other libraries. Every feature can be modified or replaced to suit your unique development workflow and feature needs. Read on to find out how.

컨트롤 추가(Add Some Control)

데이터 바인딩 (Data Binding)

데이터 바인딩은 모델이나 뷰의 변경에 따라 뷰나 모델에 자동으로 반영하는 방법이다. 이 기능은 DOM 조작으로 인한 엄청난 수고를 덜어낼 수 있다.

Data-binding is an automatic way of updating the view whenever the model changes, as well as updating the model whenever the view changes. This is awesome because it eliminates DOM manipulation from the list of things you have to worry about.

컨트롤러 (Controller)

컨트롤러는 DOM 엘리먼트 뒤의 행위이다. AngularJS는 DOM을 업데이트하고 콜백을 등록하거나 모델의 변경을 감지하는 보통의 표준 방법없이 깔끔하고 읽기 쉬운 형식으로 표현할 수 있다.

Controllers are the behavior behind the DOM elements. AngularJS lets you express the behavior in a clean readable form without the usual boilerplate of updating the DOM, registering callbacks or watching model changes.

보통 자바스크립트(Plain JavaScript)

다른 프레임워크들과는 달리 메소드 접근자의 모델을 감싸기 위해 독점적인 타입을 상속할 필요가 없다. 단지 기존의 보통 자바스크립트이면 된다. 이러한 접근 방법이 코드를 쉽게 테스트, 유지보수, 재사용 그리고 boilerplate로부터 자유롭게 한다.

Unlike other frameworks, there is no need to inherit from proprietary types; to wrap the model in accessors methods. Just plain old JavaScript here. This makes your code easy to test, maintain, reuse, and again free from boilerplate.

서버 연결(Wire up a Backend)

딥 링크 (Deep Linking)

딥 링크는 사용자가 어플리케이션에서 어디에 있는지를 나타내는 것이다. 이 것은 사용자들이 어플리케이션 내에서 위치를 북마크하거나 이메일로 링크하는데 유용할 것이다. Round trip 어플리케이션에서는 이것은 자동으로 얻을 수있는 것이었지만, AJAX 어플리케이션은 그 특성상 그렇지 않다. AngularJS는 딥 링크와 데스크톱 응용 프로그램과 같은 행동을 할 수 있습니다.

A deep link reflects where the user is in the app, this is useful so users can bookmark and email links to locations within apps. Round trip apps get this automatically, but AJAX apps by their nature do not. AngularJS combines the benefits of deep link with desktop app-like behavior.

폼 유효성 검사 (Form Validation)

클라이언트 측 폼 유효성 검사는 뛰어난 사용자 경험을 위한 중요한 부분이다. AngularJS는 JavaScript 코드를 작성하지 않고 폼에 유효성 검사 규칙을 선언 할 수 있다. 적은 코드로 작성하고, 빨리 끝낼 수 있다.

Client-side form validation is an important part of great user experience. AngularJS lets you declare the validation rules of the form without having to write JavaScript code. Write less code, go have beer sooner.

서버 커뮤니케이션 (Server Communication)

AngularJS는 XHR 위에서 서비스를 제공하는데, 코드를 매우 간단하게 해준다. 우리는 XHR을 감싸서 예외 management나 promise를 제공한다. Promise는 데이터의 리턴을 비동기적으로 처리하여 코드를 더 간단하게 한다. 리턴이 실제로 비동기적일 때 속성들을 동기적으로 할당하게 해준다.

AngularJS provides services on top of XHR, that dramatically simplify your code. We wrap XHR to give you exception management and promises. Promises further simplify your code by handling asynchronous return of data. This lets you assign properties synchronously when the return is actually asynchronous.

컴포넌트 생성(Create Components)

지시어 (Directives)

지시어들은 Angular에서만 사용 가능한 고유하고 강력한 특징이다. 지시어들은 어플리케이션만의 특유한 새로운 HTML 문법을 만들 수 있다.

Directives is a unique and powerful feature available only in Angular. Directives let you invent new HTML syntax, specific to your application.

재 사용 가능한 컴포넌트 (Reusable Components)

우리는 재사용 가능한 컴포넌트를 만드는데 지시어를 사용하고 있다. 컴포넌트를 사용하여 복잡한 DOM 구조와 CSS, 그리고 동작 수 있습니다. 따라서 어플리케이션으로 무엇을 하거나, 어플리케이션을 어떻게 개별적으로 보일지에 초점을 맞출 수 있다.

We use directives to create reusable components. A component allows you to hide complex DOM structure, CSS, and behavior. This lets you focus either on what the application does or how the application looks separately.

지역화 (Localization)

지역화는 어플리케이션에서 매우 중요한 부분이다. Angular의 로케일 인식 필터와 형태소 분석 지침이 모든 로케일에서 사용가능한 어플리케이션을 만들수 있도록 기본 요소를 제공한다.

An important part of serious apps is localization. Angular's locale aware filters and stemming directives give you building blocks to make your application available in all locales.

포함하고 주입하기(Embed and Inject)

포함가능 (Embeddable)

AngularJS는 다른 기술과 함께 사용해도 제대로 작동한다. 기존의 페이지에 원하는대로 많게든 적게든 AngularJS을 추가 할 수 있다.많은 다른 프레임워크는 전체 반영을 요구한다. 이 페이지에는 여러 AngularJS 어플리케이션이 포함되어 있습니다. AngularJS은 글로벌 상태가 없는 여러가지 어플리케이션이 단일 페이지에서 iframe을 사용하지 않고 여러 응용 프로그램을 실행시킬 수 있다. 자, view-source하고 둘러보자.

AngularJS works great with other technologies. Add as much or as little of AngularJS to an existing page as you like. Many other frameworks require full commitment. This page has multiple AngularJS applications embedded in it. Because AngularJS has no global state multiple apps can run on a single page without the use of iframes. We encourage you to view-source and look around.

주입가능 (Injectable)

AngularJS에서 의존성 주입은 선언적으로 어플리케이션 어떻게 연결되어 있는지를 기술한다. 이것은 어플리케이션이 main() 메서드를 필요하지 않음을 의미하며 보통 의존성 주입은 AngularJS의 핵심이다. 이것은 요구에 맞지 않는 어떤 컴포넌트라도 쉽게 교체될 수 있음을 뜻한다.

The dependency injection in AngularJS allows you to declaratively describe how your application is wired. This means that your application needs no main() method which is usually an unmaintainable mess. Dependency injection is also a core to AngularJS. This means that any component which does not fit your needs can easily be replaced.

테스트가능 (Testable)

AngularJS는 완벽하게 테스트가 가능하도록 설계되었다. 그것은 behavior-view 분리, mocks로 미리 번들로 제공되어 있으며, 의존성 주입을 최대한 활용하길 권장한다. 또한 AngularJS의 내부의 작동 방식을 이해하여 테스트 괴짜성을 제거하여 end-to-end 시나리오 러너가 제공됩니다.

AngularJS was designed from ground up to be testable. It encourages behavior-view separation, comes pre-bundled with mocks, and takes full advantage of dependency injection. It also comes with end-to-end scenario runner which eliminates test flakiness by understanding the inner workings of AngularJS.
